# 人员聚集检测算法说明

## 一句话概括

**让电脑自动判断：画面里有没有一群人在扎堆。**

---

## 算法做了什么（大白话版）

想象你是保安看着监控屏幕，判断"有没有人聚集"，你的思路大概是：

1. 先看画面里有哪些人 → 这是 **YOLO 模型**干的活
2. 看看这些人是不是挤在一起 → 这是**本算法**干的活
3. 如果有一小撮人靠得很近 → 标记为"聚集"

---

## 算法流程（逐步拆解）

### 第 1 步：YOLO 检测所有人

把一帧画面喂给 YOLO 模型，它会告诉你画面中每个人的位置，用**方框（bounding box）**框出来。

```
例如检测到 5 个人：
  [x1, y1, x2, y2]  表示方框左上角和右下角的坐标
```

### 第 2 步：计算每个人的"站位"

从方框算出人在画面中的**中心位置**，就像看地图时关注人的"脚下那个点"。

```
中心点 X = 方框左边 + 方框右边 的一半
中心点 Y = 方框上边 + 方框下边 的一半
```

同时记录每个人的**身体宽度**（画面中方框的宽度 = x2 - x1）。

### 第 3 步：判断"靠得近不近"

这是最关键的一步。怎么判断两个人是不是"靠在一起"？

**不用固定距离**（比如"小于 100 像素就算近"），因为：
- 近景镜头下：一个人可能有 300 像素宽，100 像素的距离其实很近
- 远景镜头下：一个人可能只有 50 像素宽，100 像素的距离其实很远

所以我们用**"一个人的宽度"作为尺子**：

```
如果两人的距离 < 1.5 个人宽 → 认为他们"靠在一起"
如果两人的距离 >= 1.5 个人宽 → 不靠在一起
```

这个"1.5"可以通过 `--distance` 参数调整。

### 第 4 步：把靠在一起的人分成一组一组

想象一个社交场景：

```
A 和 B 靠在一起（距离 < 阈值）
A 和 C 靠在一起（距离 < 阈值）
C 和 D 靠在一起（距离 < 阈值）
E 离所有人都很远
```

那么通过"朋友圈传播关系"：
- A → B → C → D 都间接联系，归为**一组**（4人）
- E 孤零零，单独**一组**（1人）

算法用 **DFS（深度优先搜索）** 实现这个过程：从任意一个人出发，找到所有和他"靠近"的人，再找这些人的"靠近的人"，直到找不到新的人为止。这一堆人就构成一个**"人群组"**。

### 第 5 步：判定哪些组算"聚集"

很简单：**一个组里人数够了就算聚集。**

```
组内有 >= 3 人 → 聚集 ✓（红色标记）
组内有 < 3 人 → 不算聚集 ✗（绿色标记）
```

这个"3"可以通过 `--min-people` 参数调整。

### 第 6 步：画到画面上

| 标记 | 含义 |
|------|------|
| 🟢 绿色方框 | 正常人员（不在聚集组中） |
| 🔴 红色方框 | 聚集人员 |
| 🟡 黄色多边形 | 聚集区域的外轮廓 |
| 🟡 xN 文字 | 这撮人有多少个 |

## 实际例子

### 场景：画面中有 6 个人

```
画面分布：
  [A]  [B]     [E]
   [C]        [F]
              [D]

A、B、C 挤在一起 → 距离判定 → {A, B, C} 为一组
D、E、F 散得很开 → 各为一组 {D}, {E}, {F}

{A, B, C} 组有 3 人 >= 3 → 聚集！
{D}, {E}, {F} 各只有 1 人 < 3 → 非聚集
```

`A、B、C` 会被画**红色框**，`D、E、F` 被画**绿色框**。

---

## 参数怎么调

| 参数 | 命令 | 默认 | 调大 | 调小 |
|------|------|------|------|------|
| 距离阈值 | `--distance` | 1.5 | 更宽容（隔得远也算靠近） | 更严格（必须贴得很近） |
| 最少人数 | `--min-people` | 3 | 需要更多人 | 2 个人也算聚集 |

### 常见场景推荐

```bash
# 地铁站/商场：空间狭窄，人一靠近就算
python crowd_detect.py --source video.mp4 --distance 1.0 --min-people 2

# 广场/公园：空间大，要很多人扎堆才算
python crowd_detect.py --source video.mp4 --distance 2.0 --min-people 5

# 默认：大多数场景够用
python crowd_detect.py --source video.mp4
```

---

## 算法的局限性

1. **遮挡**：如果一个人被另一个人挡住，YOLO 可能检测不到，聚集人数会被低估
2. **俯视角度**：从正上方往下拍时，人的"宽度"会变小，距离判断可能不准
3. **透视错觉**：3D 世界中离得很远的人，在 2D 画面里可能看起来很近
4. **只看位置**：无法区分"排队"和"聚集"，只能判断空间上是不是一堆人
