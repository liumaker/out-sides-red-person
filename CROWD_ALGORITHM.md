# 人员聚集检测算法说明

## 概述

基于 YOLOv8 目标检测结果，通过**空间距离聚类**判断视频/图片中是否存在人员聚集行为。

## 算法流程

```
视频帧 / 图片
    │
    ▼
┌─────────────────┐
│  YOLOv8 检测     │  检测所有 Person (class 0)
│  提取边界框      │  得到每个人的 (x1, y1, x2, y2)
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  计算中心点      │  cx = (x1+x2)/2,  cy = (y1+y2)/2
│  计算人体宽度    │  width = x2 - x1
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  自适应距离阈值  │  threshold = distance_factor × avg_person_width
│                  │  默认 distance_factor = 1.5
└────────┬────────┘
         │
         ▼
┌─────────────────────────┐
│  连通分量分组 (DFS)      │  两人中心距离 < threshold → 相连
│  构建邻接图，找连通分量   │  每个连通分量 = 一个"人群组"
└────────┬────────────────┘
         │
         ▼
┌─────────────────┐
│  聚集判定        │  组内人数 >= min_people → 标记为"聚集"
│                  │  默认 min_people = 3
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  可视化输出      │  红色框 = 聚集人员
│                  │  绿色框 = 正常人员
│                  │  黄色凸包 = 聚集区域轮廓
│                  │  "xN" = 该聚集组人数
└─────────────────┘
```

## 核心算法详解

### 1. 自适应距离阈值

人与人之间的"靠近"不是用固定像素值判断，而是**基于人体宽度自适应**：

```
distance_threshold = distance_factor × avg_person_width
```

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `distance_factor` | 1.5 | 距离倍率，越小越严格 |
| `avg_person_width` | 自动计算 | 当前帧所有人边界框宽度的均值 |

**为什么用自适应阈值？**
- 近景：人体像素大，距离阈值自动放大
- 远景：人体像素小，距离阈值自动缩小
- 避免因摄像头距离不同导致的误判

### 2. 连通分量分组 (DFS)

将"距离小于阈值的两人之间有边"构成的图，用深度优先搜索找出所有连通分量。

```
例：5个人，中心点距离矩阵（小于阈值的标*）

     A    B    C    D    E
  A  -    *    *    .    .
  B  *    -    .    .    .
  C  *    .    -    *    .
  D  .    .    *    -    .
  E  .    .    .    .    -

  连通分量1: {A, B, C, D}  → 4人 → 聚集 ✓
  连通分量2: {E}           → 1人 → 非聚集
```

### 3. 聚集判定

对每个连通分量：

- 人数 >= `min_people`（默认3）→ **聚集**
- 人数 < `min_people` → 非聚集

## 参数调优指南

| 参数 | 命令行 | 默认 | 调大效果 | 调小效果 |
|------|--------|------|----------|----------|
| 距离倍率 | `--distance` | 1.5 | 更容易判定为聚集（宽松） | 更难判定为聚集（严格） |
| 最少人数 | `--min-people` | 3 | 需要更多人才能算聚集 | 更少人即可算聚集 |

### 场景推荐值

| 场景 | `--distance` | `--min-people` | 说明 |
|------|-------------|----------------|------|
| 室内走廊 | 1.0 | 2 | 空间狭窄，人靠近即算聚集 |
| 室外广场 | 2.0 | 5 | 空间开阔，需更多人 |
| 商场/车站 | 1.5 | 3 | 默认值即可 |
| 高度密集场所 | 0.8 | 8 | 极度拥挤才报警 |

## 局限性

1. **遮挡问题**：严重遮挡时 YOLO 可能漏检，导致聚集人数偏低
2. **视角问题**：纯俯视视角下，人体宽度估算不准，距离阈值可能偏差
3. **2D 投影误差**：3D 空间中距离远的人可能在 2D 图像中看起来很近（透视效应）
4. **仅基于位置**：未考虑人的朝向、移动方向等行为特征，无法区分"排队"和"聚集"

## 输出说明

- **绿色框** `Person`：正常站立/行走的人员
- **红色框** `Gathering`：被判定为聚集的人员
- **黄色凸包线**：聚集区域的外轮廓
- **`xN` 标注**：该聚集组的人数
- **左上角**：总人数和聚集人数统计
- 输出视频保存在 `runs/detect/crowd/` 目录
